Chain identity gate

The selected profile fixes the expected chain, genesis, API version, and public API origin. Live /api/status is fetched and verified before signing and again before submission.

Key derivation

Purpose-separated authorities are derived inside the native shell. Seed material is imported once into the operating-system credential vault; the page keeps only an opaque wallet id and never receives stored seed bytes.

Encrypted recovery package

The package is encrypted before it leaves the native process and is bound to this wallet id, chain id, and genesis hash. Use a unique password of at least 12 characters.

Address check

Strict Bech32m: lowercase only, noos HRP, checksum verified, no error correction. Payload layout is owner-blocked (identity-v1 §2.1), so the payload stays opaque and this wallet never emits its own address yet.

Build, sign & submit transaction

This wallet supports complete note-transfer specs with one purpose-separated fee-payer account. Provide every field listed below; the shell supplies the profile chain id and fixed format version, computes the witness root and txid, fetches each input note, checks conservation/expiry/resource bytes, then emits canonical Lumen transaction and witness bytes.

Required JSON fields: expiry_height, fee_payer, fee_authorization (must be null), all six resource_limits, note_inputs, one matching account_inputs entry, object_access_list (empty), actions (empty), complete outputs, evidence_refs (empty), and one lock_reveals entry per note input. Integer values use canonical base-10 strings.

Paid Bonsai authorization

Create a profile-bound payment authorization for a remote WWM quote. The native shell checks the live chain identity first, signs only public model/profile pins and a salted prompt commitment, and never returns the wallet seed or derived secret.

Update manifest verification

An update installs only when the manifest binds this app id, chain identity, platform, arch, and subscribed channel, and the detached Ed25519 signature from the updater key verifies.

Technical provenance: NOOSPHERE research corpus. UI strings say MindChain; consensus bytes never do.
